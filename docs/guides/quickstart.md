# Quick Start Guide

Get PVE B2 Age Backup running in 10 minutes.

## Prerequisites

- **Proxmox VE** 7.x or 8.x with root (or sudo) access
- **Backblaze B2** account: create a **Bucket** (private) and an **Application Key** (Read and Write) in [Backblaze B2](https://www.backblaze.com/b2/cloud-storage.html) → Application Keys
- **Local staging space**: a directory with free space at least as large as your largest VM plus 10% (Proxmox writes the backup here temporarily; the hook then encrypts and uploads it to B2 and deletes the local file)

## Step 1: Install

The installer sets up scripts, config, systemd units, and dependencies (`rclone`, `age`, `jq`). When it generates encryption keys, **download and securely store them** (see Step 3) so you can restore backups later.

```bash
curl -fsSL https://raw.githubusercontent.com/LukasStrickler/proxmox-b2-offsite-backup/main/install.sh | sudo bash
```

## Step 2: Configure Backblaze B2 & Rclone

1.  **Backblaze**: In the [B2 dashboard](https://secure.backblaze.com/b2_buckets.htm), create a **Bucket** (private). Then go to **Application Keys** and create a key with **Read and Write** access to that bucket. If the key is restricted to one bucket, also enable **"Allow List All Bucket Names"** (rclone needs this even for single-bucket keys). Note the **Key ID** (application key ID, not the main account ID) and the **Application Key**.
2.  **Configure Rclone** (so the scripts can talk to B2):

```bash
sudo rclone config
# n (new), name: b2, type: 5 (B2), Account: <Key ID>, Key: <Application Key>, defaults for rest
```
Use the **Application Key ID** as "Account", not the main B2 Account ID.  
*Tip: When asked about advanced config, set `b2-hard-delete` to `false`.*

## Step 3: Configure Backup System

### 3.1 Edit Config
```bash
sudo nano /etc/pve-b2-age-backup/config.env
```
Ensure these are set:
```bash
RCLONE_REMOTE="b2:YOUR_BUCKET_NAME"
DUMPDIR="/backup/vzdump"
# Keys are auto-generated by installer at these paths:
AGE_RECIPIENTS="/etc/pve-b2-age-backup/recipients.txt"
AGE_IDENTITY="/etc/pve-b2-age-backup/age.key" 
```

**Critical — store your keys:** Download and securely store `age.key` (and optionally `recipients.txt`) to a safe, offline location (e.g. encrypted USB, password manager) **before** relying on backups. Without the private key you cannot restore backups or get them working again after a disaster.

### 3.2 Create the staging directory
Proxmox will write backup files here; the hook encrypts and uploads them to B2, then deletes the local file. You need enough space for **one** full backup (your largest VM + 10%).

**Recommended (ZFS):**
```bash
sudo zfs create -o mountpoint=/backup/vzdump -o quota=120G rpool/backup-vzdump
```
*Or for directory: `mkdir -p /backup/vzdump && chmod 755 /backup/vzdump`*

### 3.3 Add the staging directory to Proxmox
So Proxmox can use it as backup storage: **Datacenter → Storage → Add → Directory**
- **ID**: `backup-vzdump` (or any name you like)
- **Directory**: `/backup/vzdump` (must match `DUMPDIR` in config)
- **Content**: check **VZDump backup file**

## Step 4: Create a backup job
Create one backup job per VM or container. The **hook script** is what runs after each backup to encrypt and upload the file to B2.

**Datacenter → Backup → Add**
1.  **Storage**: `backup-vzdump` (the staging directory you added)
2.  **Selection**: Choose **one** VM or CT (do not select "All"—create a separate job for each)
3.  **Mode**: `Snapshot`
4.  **Hook script**: `/usr/local/sbin/pve-b2-age-hook.sh`

## Step 5: Enable timers
Enable automatic **prune** (removes old backups on B2) and **host config** backup (weekly backup of `/etc` and related config):
```bash
sudo systemctl enable --now pve-b2-age-prune.timer pve-b2-age-hostconfig.timer
```

## Step 6: Verify & Test
**1. Run Pre-flight Check (New):**
```bash
sudo pve-b2-age-check.sh
```
*Fix any errors reported before proceeding.*

**2. Run Manual Backup:**
Select job in UI → **Run now**. Watch **Task History**.

**3. Check Logs:**
```bash
sudo tail -f /var/log/pve-b2-age.log
```

**4. Verify Cloud Files:**
```bash
sudo rclone lsf b2:YOUR_BUCKET_NAME/proxmox/$(hostname -s)/daily/
```

**5. Test restore (important):**  
Run a restore to confirm backups and keys work. Replace the placeholders with real values from the list:
```bash
# List backups (note a filename from the output)
sudo pve-b2-age-list.sh daily

# Restore one backup to a new VM/CT ID (e.g. 999) that does not exist yet
# Replace BACKUP_FILENAME with a file from the list, e.g. vzdump-qemu-101-2026_02_17-02_00_01.vma.zst.age
# Replace 999 with an unused VM/CT ID
sudo pve-b2-age-restore.sh daily BACKUP_FILENAME 999
```

## Terms you might see

| Term | Meaning |
|------|--------|
| **Staging** | The folder (`DUMPDIR`) where Proxmox writes the backup file before the hook uploads it. Only one backup at a time needs to fit here. |
| **Hook** | The script Proxmox runs after a backup (encrypt + upload to B2 + delete local file). |
| **Prune** | A scheduled job that deletes old backups on B2 so you keep only the last N daily/monthly. |
| **Host config** | A weekly backup of `/etc`, `/var/lib/pve-cluster`, and `/root` to B2 (for disaster recovery). |
